# Agent Guidelines — 文头字脑 (still)

A Wails v2 application (Go backend + vanilla JS frontend) for a Chinese literary game about collecting textual fragments.

---

## 1. Build & Run Commands

### Wails Commands (Root Directory)

| Command | Description |
|---------|-------------|
| `wails dev` | Run in development mode with hot reload |
| `wails build` | Build production binary/app |
| `wails build -platform darwin` | Build macOS app specifically |

### Frontend Commands (frontend/ directory)

| Command | Description |
|---------|-------------|
| `npm install` | Install frontend dependencies |
| `npm run dev` | Run Vite dev server (used by wails dev) |
| `npm run build` | Build frontend assets to `frontend/dist/` |
| `npm run preview` | Preview built frontend |

### Go Commands

| Command | Description |
|---------|-------------|
| `go build ./...` | Build all Go packages |
| `go mod tidy` | Clean up go.mod dependencies |
| `go vet ./...` | Run Go vet for static analysis |
| `go test ./...` | Run tests (none currently exist) |

**Single test**: `go test -v -run TestName ./...`

## 3. Go Code Style

**Go Version**: 1.23

### Formatting
- Use `gofmt` (standard formatter) — no custom config
- Tab indentation, standard Go spacing

### Imports
```go
import (
    "context"
    "fmt"
    "os"
    "path/filepath"
    "time"

    "github.com/wailsapp/wails/v2"
    "github.com/wailsapp/wails/v2/pkg/options"
    "github.com/wailsapp/wails/v2/pkg/options/assetserver"

    _ "github.com/mattn/go-sqlite3"
)
```
- Standard library first, then third-party
- Blank import `_ "github.com/mattn/go-sqlite3"` for driver side-effect init

### Error Handling
- Wrap errors with context: `fmt.Errorf("operation failed: %w", err)`
- Always check and return errors from public methods
- Print to stdout in startup/shutdown (not logging library)

```go
// Good
if err := os.MkdirAll(dataDir, 0755); err != nil {
    fmt.Printf("Failed to create data dir: %v\n", err)
    return
}

// Good - public method
func (a *App) SaveFragment(frag Fragment) error {
    return dbSaveFragment(frag)
}
```

### Naming Conventions
- **Structs**: PascalCase (`Fragment`, `Creation`, `PaginatedResult`)
- **Fields**: PascalCase with JSON tags for frontend compatibility
- **DB columns**: snake_case (SQLite convention)

```go
type Fragment struct {
    ID          int64   `json:"id"`
    Title       string  `json:"title"`
    FullTitle   string  `json:"fullTitle"`
    Description string  `json:"description"`
    CollectedAt string  `json:"collectedAt"`
    // Position data (generated by seeded RNG)
    PositionX   float64 `json:"positionX"` // 5-65%
    PositionY   float64 `json:"positionY"` // 8-83%
    Angle       float64 `json:"angle"`     // -15 to 15 degrees
}
```

### Wails Binding
- App struct methods are exposed to JS via `Bind: []any{app}` in main.go
- All bound methods must be on the `App` struct (exported or unexported)
- Frontend calls: `window.go.main.App.MethodName()`

### Database
- SQLite with WAL mode: `file:path?_journal_mode=WAL&_busy_timeout=5000&_foreign_keys=ON`
- Single connection only: `SetMaxOpenConns(1)`, `SetMaxIdleConns(1)`
- Always `defer rows.Close()` after Query operations

## 4. TypeScript Code Style

### Comments
- Avoid comments unless necessary
- If needed, write comments in **English**
- Self-documenting code is preferred over explanatory comments

### Async Patterns
- Prefer `async/await`
- Always handle `.catch()` on Promise chains

```typescript
async function init(): Promise<void> {
    try {
        const result = await (window as any).go.main.App.LoadCollected(1, 10);
        if (result && result.items) {
            collectedFragments = result.items;
        }
    } catch (e) {
        console.warn('LoadCollected failed:', e);
    }
}
```

### Data Flow (Go-first)
- Fragment positioning (X, Y, angle) generated by Go's seeded RNG
- TS receives pre-positioned fragments from `GetFragments`
- Client-side filtering of collected fragments done in Go SQL query
- TS only handles rendering and DOM events

### Wails Runtime
```javascript
// Logging (output to wails dev terminal)
import { LogDebug, LogInfo, LogError } from './wailsjs/runtime/runtime';

// Events (Go <-> JS communication)
import { EventsOn, EventsEmit } from './wailsjs/runtime/runtime';

EventsOn('backend:debug', (data) => { console.log('From Go:', data); });
EventsEmit(ctx, "backend:debug", { status: "ok" });
```

### DOM Patterns
- Query selectors cached in functions, not globally
- Event listeners attached in setup functions
- Modal/overlay toggled via CSS class `.is-active`

```typescript
function closeModal(): void {
    const overlay = document.querySelector('.fragment-overlay');
    if (overlay) overlay.classList.remove('is-active');
    syncFragmentsVisibility();
}
```

### Window Exports
- Expose functions/modules via `window.*` property with type assertions
```typescript
(window as unknown as { _audio: AudioSystem })._audio = audio;
(window as unknown as { _renderNarrative: () => void })._renderNarrative = renderNarrative;
```

## 5. CSS Code Style

**No CSS preprocessor, no PostCSS** — vanilla CSS with custom properties.

### Structure
- CSS custom properties (design tokens) at `:root`
- Sections marked with `/* ── N. Section Name ─────────────────────────────── */`
- BEM-ish naming: `.fragment-detail__text`, `.narrative-item__word`

### Custom Properties (Design Tokens)
```css
:root {
    /* Colors */
    --color-ink: #2c2c2c;
    --color-paper: #f5f0e8;
    --color-accent: #8b4513;

    /* Typography */
    --font-display: "Ma Shan Zheng", "Noto Sans SC", sans-serif;
    --font-body: "Noto Sans SC", sans-serif;

    /* Spacing (4px base) */
    --space-1: 0.25rem;
    --space-4: 1rem;
    --space-8: 2rem;

    /* Transitions */
    --ease-default: 0.25s ease;
    --ease-spring: 0.3s cubic-bezier(0.34, 1.56, 0.64, 1);
}
```
